# Purpose
Integration test directory for the be-iis-installer project. Intended for tests that validate installer functionality across multiple modules or subsystems.

# Files
No tests or scripts are present by default in this directory.

# Usage
Place integration test scripts or configuration files here to verify the interaction of multiple installer components (e.g., overlays, scripts, or services).

# Notes
- Test scripts should use standard exit codes.
- Keep tests self-contained and document dependencies within each script.
- Refer to sibling and upstream test directories for specific module or overlay tests.
